# Firebase 下一步设置教程

## ✅ 当前状态
Firebase 项目 `telegram-bot-new` 已创建成功！

## 🚀 步骤 1: 启用 Firestore 数据库

### 操作步骤：

1. **在 Firebase 控制台首页，点击「建立後端」卡片**
   - 位置：页面中间偏左的卡片（見截图）
   - 文字：「建立後端」

2. **或者在左侧导航栏找到「Firestore Database」**
   - 点击左侧菜单栏（三道横线图标）
   - 找到「建構」分类下的「Firestore Database」

3. **点击「建立資料庫」**

4. **选择安全规则：
   - ✅ 选择「測試模式」（允許所有讀寫）
   - 稍后可改为生产模式

5. **选择数据库位置：
   - 推荐选择：`asia-east1`（台灣）或 `asia-southeast1`（新加坡）
   - 点击「啟用」

6. **等待数据库创建完成**（约 30 秒）

---

## 🚀 步骤 2: 生成服务账户密钥

### 操作步骤：

1. **点击左上角的齿轮图标「⚙️ 設定」**
   - 位于项目名称旁边

2. **选择「專案設定」**

3. **切换到「服務帳戶」标签页**
   - 点击顶部的「服務帳戶」

4. **点击「產生新的私鑰」按钮**
   - 选择「Node.js」作为语言（如果询问）
   - 点击蓝色的「產生新的私鑰」按钮

5. **下载 JSON 文件**
   - 浏览器会自动下载一个 `.json` 文件
   - 文件名称类似：`telegram-bot-new-firebase-adminsdk-xxxxx.json`

6. **重命名并移动文件**
   - 将文件重命名为：`service-account-key.json`
   - 移动到项目目录：`C:\Users\t2903\WorkBuddy\2026-06-04-15-29-30\telegram-bot\config\`

## ⚠️ 常见问题

### 问题 1：找不到「Firestore Database」
**解决**：点击左侧菜单栏（三道横线），在「建構」分类下找到

### 问题 2：「產生新的私鑰」按钮灰色
**解决**：确认已选择「服務帳戶」标签，且有权限（您是项目所有者）

### 问题 3：下载的文件找不到
**解决**：检查浏览器的「下载」文件夹，文件名称包含 `firebase-adminsdk`

### 问题 4：机器人显示 Firebase 连接错误
**解决**：
- 确认 `service-account-key.json` 已放入 `config/` 目录
- 确认文件名完全匹配（区分大小写）
- 重新启动机器人
